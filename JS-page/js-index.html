Array.map()
This method produces another array to perform the function given on each array element, this method will not execute if there are no values inside the array.

Example:

var arr = [5, 4, 3, 2, 1];
var arr2 = arr.map(mySample);
document.getElementById("example").innerHTML = arr2;

function mySample(value) {
Return value * 5;
}

Result:

25,20,15,10,5

Array.filter() / every() / some()
The filter() method produces and stores elements in another array that meets the standards of a function given. every() almost do the same functionalities, except, it returns a boolean value and checks if all elements inside the array complies with the given function. some() also has the same functionalities with every() but it only concerns other elements in the array.

Example:

var arr = [5, 4, 3, 2, 1];
var less3 = arr.map(mySample);
document.getElementById("example").innerHTML = less3;

function mySample(value) {
Return value < 3;
}

Result:

2,1 [ filter() ]
false [ every() - it returned false because not all values inside the array are less than 3 ]
true [ some() - it returned true because some elements in the array are less than 3]


Array.reduce()
This method produces a value that consists of all the elements inside the array, this method follows a left-to-right rule when complying to a function. reduceRight() is the same as reduce() but it follows a right-to-left rule.

Example:

var arr = [5, 4, 3, 2, 1];
var diff= arr.reduce(mySample);
document.getElementById("example").innerHTML = “The difference is ” + diff;

function mySample(total, value) {
Return total - value;
}

Result:

-5 [ reduce() ]
-13 [ reduceRight() ]

Array.indexOf() / lastIndexOf() / find()
This indexOf() method looks for an element and returns the position of that element. lastIndexOf() searches an element from the end of the array. find() looks for an element that passes the requirements of the function given.

JavaScript Arithmetic

An arithmetic operator in JavaScript includes a variable,literal, or expressions that will return a numeric value from an operand (the numbers that are used in between the operator).

Example:
var x = 75 + 75; (literals)
var x = y + z; (variables)
var x = (99 + 65) * y; (expressions)


Types of Arithmetic Operators:
1. Addition ( + )
2. Subtraction ( - )
3. Multiplication ( * )
4. Division ( / )
5. Modulus ( % ) - remainder
6. Increment ( ++ ) - increases a variable by 1
7. Decrement ( -- ) - decreases a variable by 1
8. Exponentiation ( ** ) - this raises the first operand to the second operand; this shows the same results as “Math.pow(z,y)”

JavaScript Assignment

An assignment operator basically assigns a value from the left operand to the right operand.

TYPES of Assignment Operators:
1. =
2. += ; a = a + b
3. -= ; a = a - b
4. *= ; a = a * b
5. /= ; a = a / b
6. %= ; a = % b
7. &= ; a = a & b (AND)
8. |= ; a = a | b (OR)
9. ^= ; a = a ^ b (XOR)
10. <<= ; a = a << b (left shift)
11. >>= ; a = a >> b (right shift)
12. >>>= ; a = a >>> b (unsigned right shift)

Bitwise OR, XOR, AND - this uses binary representation of both operands.

Example:
var meow = 9;
meow &= 1;

9: 0000000000001001
1: 0000000000000001

AND = 1 OR = 0000000000001001
XOR = 0000000000001000

Left Shift, Right Shift, Unsigned Right Shift - this moves the specified bits and assigns the result to the variable.

JavaScript Comparisons

Comparison operators are used to identify the difference between variables or values.

Types of Comparison Operators:

2. Equal value and equal type ( === )
3. Not equal ( != )
4. Not equal / not equal type ( !== )
5. Greater than ( > )
6. Less than ( < )
7. Greater than or equal to ( >= )
8. Less than or equal to ( <= )


Logical operators are associated with Boolean values.

Types of Logical Operators:

JavaScript Data Types

Data types are the numbers, objects, strings, arrays, and others. Data types helps in operating with variables.
Data types in JavaScript are Dynamic, meaning, a variable can hold different types of variables. A variable without any value is called “undefined”, but, this is different with a variable with an “empty value” (ex: var name = “”;), an empty value is considered legal and has a data type.
Null, this is supposedly considered to be “nothing” but in JavaScript, Null is used to define an object is empty. What is the difference between Null and Undefined? A Null is still an object even if it has no value, undefined is still undefined whether having a value or not.
